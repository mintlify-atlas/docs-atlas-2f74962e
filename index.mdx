gosec is a static analysis tool for Go that inspects your source code for security vulnerabilities. It scans the Go AST and SSA representations to detect issues ranging from hardcoded credentials and SQL injection risks to weak cryptography and path traversal. With over 50 built-in rules, taint analysis, and support for multiple output formats including SARIF, gosec integrates seamlessly into local development, CI pipelines, and GitHub Actions.

## Why gosec?

gosec catches security issues that typical linters miss — problems that lead to real vulnerabilities in production Go services.

## Get started

<Steps>
  <Step title="Install gosec">
    Install the latest version using Go modules:

    ```bash
    go install github.com/securego/gosec/v2/cmd/gosec@latest
    ```
  </Step>
  <Step title="Scan your project">
    Run gosec against all packages in your module:

    ```bash
    gosec ./...
    ```
  </Step>
  <Step title="Review results">
    gosec prints findings to stdout with severity, confidence, and CWE mapping. Exit code `1` means issues were found; `0` means clean.
  </Step>
  <Step title="Integrate with CI">
    Add gosec to your GitHub Actions workflow and upload SARIF results to GitHub code scanning for persistent tracking.

    See the [GitHub Actions integration guide](/integrations/github-actions) for a complete workflow example.
  </Step>
</Steps>

## Rule categories

gosec organizes its rules into seven categories covering the most common Go security issues:

| Category | Rules | Description |
|----------|-------|-------------|
| G1xx | G101–G124 | General secure coding: hardcoded credentials, HTTP hardening, cookie security |
| G2xx | G201–G204 | Injection: SQL construction, template escaping, command execution |
| G3xx | G301–G307 | Filesystem: file permissions, path traversal, zip extraction |
| G4xx | G401–G408 | Crypto & TLS: weak algorithms, bad TLS config, hardcoded IVs |
| G5xx | G501–G507 | Import blocklist: dangerous crypto packages |
| G6xx | G601–G602 | Runtime safety: range aliasing, slice bounds |
| G7xx | G701–G709 | Taint analysis: SQL, command, path, SSRF, XSS, log, SMTP injection |
