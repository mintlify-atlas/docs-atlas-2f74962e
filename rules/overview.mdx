gosec ships with over 50 rules organized into seven categories. Rules are identified by a `Gxxx` ID and each is mapped to a CWE (Common Weakness Enumeration) identifier. By default, all rules run against the scanned packages — use `-include` or `-exclude` flags to customize the active set.

## Implementation types

gosec uses three analysis techniques:

| Type | Description | Location |
|------|-------------|----------|
| **AST** | Pattern-based checks on Go syntax trees | `rules/` package |
| **SSA** | Whole-program analysis using static single assignment form | `analyzers/` package |
| **Taint** | Source-to-sink data flow tracking | `analyzers/` via taint framework |

## Selecting rules

```bash
# Run only specific rules
gosec -include=G101,G203,G401 ./...

# Exclude a specific rule
gosec -exclude=G303 ./...

# Exclude rules for specific paths
gosec --exclude-rules="cmd/.*:G204,G304" ./...
```
