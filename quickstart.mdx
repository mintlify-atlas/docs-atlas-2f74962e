title: "Get started with gosec in two minutes"
description: "Scan your first Go project with gosec in under two minutes. Learn to read findings, interpret severity and confidence scores, and export reports."
---

gosec requires no configuration to get started. Point it at any Go module and it will scan all packages, printing findings to stdout.

## Scan your first project

<Steps>
  <Step title="Navigate to your Go project">
    ```bash
    cd /path/to/your/go/project
    ```
    The directory must contain a `go.mod` file. gosec uses Go modules to load packages.
  </Step>
  <Step title="Run gosec">
    Scan all packages in the module recursively:

    ```bash
    gosec ./...
    ```

    To scan a single package:

    ```bash
    gosec github.com/example/mypackage
    ```
  </Step>
  <Step title="Read the output">
    gosec prints each finding with its rule ID, severity, confidence, file location, and CWE mapping:

    ```
    [/path/to/file.go:42] - G101 (CWE-798): Potential hardcoded credentials (Confidence: LOW, Severity: HIGH)
      > 42: password := "hunter2"
    ```

    The summary at the end shows total issues found, files scanned, and lines of code checked.
  </Step>
</Steps>

## Understanding results

Each finding includes:

| Field | Description |
|-------|-------------|
| **File:line** | Exact location of the issue |
| **Rule ID** | The gosec rule that fired (e.g., G101) |
| **CWE** | Common Weakness Enumeration identifier |
| **Description** | What the rule checks for |
| **Severity** | HIGH, MEDIUM, or LOW |
| **Confidence** | HIGH, MEDIUM, or LOW |

## Exit codes

| Code | Meaning |
|------|---------|
| `0` | Scan completed with no unsuppressed findings or errors |
| `1` | At least one unsuppressed finding or processing error |

Use `-no-fail` to always exit `0` — useful in CI when you want to collect results without failing the build:

```bash
gosec -no-fail ./...
```

## Common scan patterns

```bash
# Scan all packages and write JSON report
gosec -fmt json -out results.json ./...

# Scan all packages and write SARIF report for GitHub code scanning
gosec -fmt sarif -out results.sarif ./...

# Only report medium severity or higher findings
gosec -severity medium ./...

# Run only specific rules
gosec -include=G101,G203,G401 ./...

# Exclude a specific rule
gosec -exclude=G303 ./...

# Include test files in the scan (excluded by default)
gosec -tests ./...

# Exclude generated files
gosec -exclude-generated ./...
```
