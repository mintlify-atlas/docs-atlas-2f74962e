---
title: "Scanning Go packages with gosec"
description: "Control what gosec scans with flags for test files, generated code, directories, build tags, severity thresholds, and concurrency."
---

gosec analyzes Go source code by scanning the Go AST and SSA representation. You pass one or more package patterns as arguments, and gosec applies all enabled rules to every file it loads. The flags below let you narrow or widen the scope of each scan—from a single package to an entire module, with fine-grained control over which files, directories, and build configurations are included.

## Specifying what to scan

The most common pattern is `./...`, which tells gosec to scan every package in the current module recursively.

```bash
# Scan all packages in the current module
gosec ./...

# Scan a single package
gosec github.com/example/project

# Scan a specific directory (non-recursively)
gosec $GOPATH/src/github.com/example/project
```

You can also use the `-r` flag, which appends `./...` to the target directory automatically:

| Flag | Default | Description |
|------|---------|-------------|
| `-r` | `false` | Appends `./...` to the target directory |

```bash
gosec -r
```

## Including and excluding files

By default, gosec skips test files (`_test.go`) and the `vendor/` directory. The following flags adjust that behavior.

| Flag | Default | Description |
|------|---------|-------------|
| `-tests` | `false` | Scan test files (`_test.go`) in addition to regular source files |
| `-exclude-generated` | `false` | Skip files that contain the standard generated-code comment |
| `-exclude-dir` | `vendor`, `.git/` | Exclude a directory from the scan (repeatable) |

```bash
# Include test files
gosec -tests ./...

# Exclude files with "Code generated ... DO NOT EDIT." headers
gosec -exclude-generated ./...

# Exclude one or more directories
gosec -exclude-dir=rules -exclude-dir=cmd ./...
```

<Note>
`vendor/` and `.git/` are always excluded by default. You do not need to specify them with `-exclude-dir`.
</Note>

The `-exclude-generated` flag matches the standard Go generated-file comment:

```go
// Code generated by some generator DO NOT EDIT.
```

## Build tags

gosec passes build tags to the Go toolchain when loading packages. Provide them as a comma-separated list:

| Flag | Default | Description |
|------|---------|-------------|
| `-tags` | _(none)_ | Comma-separated list of Go build tags to apply during analysis |

```bash
gosec -tags debug,ignore ./...
```

## Severity and confidence thresholds

Use these flags to suppress findings below a certain severity or confidence level. Issues below the threshold are still detected but filtered out of the report and do not affect the exit code.

| Flag | Default | Valid values | Description |
|------|---------|-------------|-------------|
| `-severity` | `low` | `low`, `medium`, `high` | Filter out findings with severity lower than this value |
| `-confidence` | `low` | `low`, `medium`, `high` | Filter out findings with confidence lower than this value |

```bash
# Only report medium or higher severity
gosec -severity medium ./...

# Only report high-confidence findings
gosec -confidence high ./...

# Combine both thresholds
gosec -severity medium -confidence medium ./...
```

## Concurrency

gosec processes packages in parallel. By default it uses all available CPU cores.

| Flag | Default | Description |
|------|---------|-------------|
| `-concurrency` | `runtime.NumCPU()` | Number of packages to analyze concurrently |

```bash
gosec -concurrency 4 ./...
```

## Output verbosity

| Flag | Default | Description |
|------|---------|-------------|
| `-quiet` | `false` | Suppress all output unless findings are detected |
| `-terse` | `false` | Show only the results and summary, discarding all log messages |

```bash
# No output on clean scans
gosec -quiet ./...

# Minimal output—results and summary only
gosec -terse ./...
```

<Tip>
`-terse` is useful in CI pipelines where you want compact output without suppressing the findings themselves.
</Tip>

## Audit mode

Audit mode enables additional checks that may be too noisy for day-to-day analysis. It can be set via a flag or a [configuration file](/usage/filtering).

| Flag | Default | Description |
|------|---------|-------------|
| `-enable-audit` | `false` | Enable audit mode, which turns on additional checks |

```bash
gosec -enable-audit ./...
```

## No-fail mode

By default, gosec exits with code `1` when it finds unsuppressed issues. Use `-no-fail` to always return `0`, which is useful when you want to collect results without blocking a pipeline.

| Flag | Default | Description |
|------|---------|-------------|
| `-no-fail` | `false` | Exit with code `0` even if issues are found |

```bash
gosec -no-fail -fmt sarif -out results.sarif ./...
```

## Sorting

| Flag | Default | Description |
|------|---------|-------------|
| `-sort` | `true` | Sort findings by severity before reporting |

Sorting is enabled by default. Disable it if you need findings in file-order:

```bash
gosec -sort=false ./...
```

## Version information

| Flag | Default | Description |
|------|---------|-------------|
| `-version` | `false` | Print version, git tag, and build date, then exit with code `0` |

```bash
gosec -version
```

## Exit codes

| Code | Meaning |
|------|---------|
| `0` | Scan finished with no unsuppressed findings or errors |
| `1` | At least one unsuppressed finding or processing error |

Use `-no-fail` to always return `0` regardless of findings.
