---
title: "Filtering gosec findings by rule, severity, and path"
description: "Include or exclude specific rules, filter by severity and confidence levels, and exclude directories or generated files from gosec scans."
---

gosec provides several independent mechanisms for controlling which findings appear in a report. You can select rules by ID, set minimum severity and confidence thresholds, exclude entire directories, skip generated files, and apply path-specific rule suppression through a configuration file or a CLI flag. These mechanisms can be combined freely.

## Selecting rules by ID

By default, gosec runs every available rule. Use `-include` to run only a specific subset, or `-exclude` to remove individual rules from an otherwise complete run.

| Flag | Default | Description |
|------|---------|-------------|
| `-include` | _(all rules)_ | Comma-separated list of rule IDs to enable. All other rules are disabled. |
| `-exclude` | _(none)_ | Comma-separated list of rule IDs to disable. All other rules remain enabled. |

```bash
# Run only G101, G203, and G401
gosec -include=G101,G203,G401 ./...

# Run everything except G303
gosec -exclude=G303 ./...
```

<Note>
`-include` and `-exclude` affect both AST-based rules and SSA-based analyzers. You can mix rule IDs from any category in the same flag value.
</Note>

## Filtering by severity and confidence

These flags suppress findings that fall below the specified threshold. Filtered issues are not included in the report and do not contribute to a non-zero exit code.

| Flag | Default | Valid values | Description |
|------|---------|-------------|-------------|
| `-severity` | `low` | `low`, `medium`, `high` | Hide findings with severity below this level |
| `-confidence` | `low` | `low`, `medium`, `high` | Hide findings with confidence below this level |

```bash
# Show only medium or higher severity
gosec -severity medium ./...

# Show only high-confidence findings
gosec -confidence high ./...

# Apply both thresholds simultaneously
gosec -severity medium -confidence medium ./...
```

## Excluding directories

gosec automatically excludes `vendor/` and `.git/`. Use `-exclude-dir` to exclude additional directories. The flag can be specified multiple times.

| Flag | Default | Description |
|------|---------|-------------|
| `-exclude-dir` | `vendor`, `.git/` | Exclude a directory from the scan (repeatable) |

```bash
gosec -exclude-dir=rules -exclude-dir=cmd ./...
```

## Excluding generated files

gosec can skip files that carry the standard Go generated-file header:

```
// Code generated by some generator DO NOT EDIT.
```

| Flag | Default | Description |
|------|---------|-------------|
| `-exclude-generated` | `false` | Skip files with the standard generated-code comment |

```bash
gosec -exclude-generated ./...
```

## Including test files

Test files (`_test.go`) are excluded by default. Enable scanning them with `-tests`.

| Flag | Default | Description |
|------|---------|-------------|
| `-tests` | `false` | Include `_test.go` files in the scan |

```bash
gosec -tests ./...
```

## Path-based rule exclusions

Large repositories with multiple components may need different rules for different paths. The `--exclude-rules` flag (or an equivalent configuration file entry) maps path patterns to rule lists.

### CLI flag

```bash
# Suppress G204 and G304 for files under cmd/
gosec --exclude-rules="cmd/.*:G204,G304" ./...

# Suppress all rules for files under scripts/
gosec --exclude-rules="scripts/.*:*" ./...

# Multiple path-rule pairs, separated by semicolons
gosec --exclude-rules="cmd/.*:G204,G304;test/.*:G101" ./...
```

The format for each entry is `<path-regex>:<rule1>,<rule2>`. Use `*` to suppress all rules for a path.

| Field | Type | Description |
|-------|------|-------------|
| `path` | string (regex) | Regular expression matched against file paths |
| `rules` | list | Rule IDs to exclude for matching paths. Use `*` for all rules. |

### Configuration file

The same path-based exclusions can be declared in a JSON configuration file passed with `-conf`:

```json
{
  "exclude-rules": [
    {
      "path": "cmd/.*",
      "rules": ["G204", "G304"]
    },
    {
      "path": "scripts/.*",
      "rules": ["*"]
    }
  ]
}
```

```bash
gosec -conf config.json ./...
```

When both the CLI flag and the config file define exclusions, they are merged. CLI rules take precedence over config file rules for the same path.

## Global configuration file

A JSON configuration file can set global options for nosec handling and audit mode. Pass the file path with `-conf`.

```json
{
  "global": {
    "nosec": "enabled",
    "audit": "enabled"
  }
}
```

| Key | Description |
|-----|-------------|
| `nosec` | When set to `"enabled"`, overrides all `#nosec` directives in the codebase (equivalent to `-nosec=true`) |
| `audit` | When set to `"enabled"`, activates audit mode for additional checks (equivalent to `-enable-audit`) |

```bash
gosec -conf config.json ./...
```
